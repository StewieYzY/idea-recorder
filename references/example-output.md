## 1. 基于对话触发的 Idea 自动归档系统

**记录时间：** 14:32

### ✅ 确定性结论

在 Claude Code 等 agent 环境中，当用户说出"记录想法"类话语时，自动触发 idea 归档流程：在项目根目录下维护 `idea/` 文件夹，以日期为文件名（`YYYY-MM-DD.md`），将当次对话中的核心想法结构化写入，同一天多条 idea 追加写入同一文件。

### 🧠 精华思考与辩论

**核心取舍：按天归档 vs 按 idea 单独建文件**
讨论了两种方案：每个 idea 独立一个文件（便于单独引用）vs 同天 idea 合并到一个文件（减少文件碎片化）。最终选择按天归档，理由是：日常工作流中 idea 产生频率不高，按天查找符合人类的时间记忆习惯；单独文件反而造成 `idea/` 目录散乱，不便于回顾。

**关键判断：只记录"确认的结论"，不记录草稿**
AI 输出的建议在被人明确认可之前不应写入。这避免了"归档了一堆 AI 泡沫观点"的问题，确保每条 idea 文件都是高信噪比的知识资产。

### 🎯 需求背景与应用场景

**背景：**
在 Claude Code 等长对话工作流中，经常出现有价值的想法和讨论，但这些内容随着对话结束就消失了。没有低摩擦的方式把"灵光一现"的想法留存下来，导致很多好点子被遗忘。

**应用场景：**
- 开发过程中讨论出新功能方向，想留存决策逻辑
- 产品讨论中产生了新的设计思路，想记录下来供后续参考
- 解决问题时发现了某个通用方法论，值得沉淀

**目标：**
让"记录想法"的行为从主动整理变为自然触发，降低知识沉淀的摩擦，让 `idea/` 目录成为项目中有价值的思考资产库。

### 🔧 具体解法

实现为一个 Claude Skill（SKILL.md），部署在 Claude Code 环境中：

1. 监听用户的触发信号（"记录"、"存下来"等话语）
2. 自动定位项目根目录（优先 git root）
3. 获取系统当前日期，确定写入文件路径 `idea/YYYY-MM-DD.md`
4. 回顾当前对话，按六维模板提炼内容（结论、精华思考、背景场景、解法、待办）
5. 判断文件是否存在：新建或追加，保持幂等
6. 写入后向用户简短确认文件路径和标题

## 2. 用 Git Hooks 自动触发 idea 文件的 commit

**记录时间：** 15:47

### ✅ 确定性结论

在项目中配置 post-commit hook，当 `idea/` 目录下有文件变更时，自动生成一条格式化的 commit message（如 `docs(idea): add idea on 2025-01-15`），减少手动 commit 的心智负担。

### 🧠 精华思考与辩论

讨论了是否应该让 idea 记录触发自动 commit。主要顾虑是"自动 commit 可能打乱工作流"。最终判断：用 post-write hook（而非强制 commit）更合适，将 commit 权限保留给用户，hook 只做提示。

### 🎯 需求背景与应用场景

**背景：**
idea 文件写入后经常忘记提交，导致本地有记录但远程仓库没有，无法在多设备间同步。

**应用场景：**
使用 git 管理项目且希望 idea 记录随代码同步到远端的场景。

**目标：**
让 idea 记录自动进入版本控制，无需额外操作。

### 🔧 具体解法

在 `.git/hooks/post-commit` 中添加脚本：检测本次 commit 是否包含 `idea/` 路径的变更，若是则在终端输出提示（不强制执行任何操作）。更激进的版本是在 `idea/` 文件写入后立即 `git add` + `git commit`，但需用户显式开启此选项。
